# 通訊服務登入後啟動功能測試說明

## 功能概述
本功能確保 SCADA 系統的通訊服務（Modbus 通訊、MQTT 發布、MQTT 訂閱）只有在使用者成功登入後才開始執行，提升系統安全性。

## 主要元件

### 1. CommunicationServiceManager.cs
**位置**: `ScadaEngine.Web\Services\CommunicationServiceManager.cs`
**功能**: 
- 管理通訊服務的生命週期
- 提供服務啟動/停止介面
- 檢查服務狀態

**核心方法**:
- `StartCommunicationServicesAsync(szUserName)`: 啟動所有通訊服務
- `StopCommunicationServicesAsync(szReason)`: 停止所有通訊服務
- `GetServiceStatus()`: 取得服務狀態資訊

### 2. LoginController.cs 修改
**位置**: `ScadaEngine.Web\Features\Authentication\Controllers\LoginController.cs`
**修改內容**:
- 注入 `CommunicationServiceManager`
- 登入成功後自動啟動通訊服務
- 登出時自動停止通訊服務

### 3. Program.cs 修改
**位置**: `ScadaEngine.Web\Program.cs`
**修改內容**:
- 註冊通訊服務為 Singleton（但不自動啟動）
- 註冊 `CommunicationServiceManager` 服務

### 4. HomeController.cs 與主頁視圖
**修改內容**:
- 主頁顯示通訊服務即時狀態
- 狀態包含：服務啟動狀態、MQTT 連線狀態、Modbus 採集狀態

## 運作流程

### 登入流程:
1. 使用者提交登入表單
2. 驗證帳號密碼
3. 建立認證 Cookie
4. **自動啟動通訊服務** ⬅ 新功能
5. 導向主頁面

### 登出流程:
1. 使用者點擊登出
2. **自動停止通訊服務** ⬅ 新功能
3. 清除認證 Cookie
4. 導向登入頁面

### 主頁顯示:
- 通訊服務啟動狀態
- MQTT 發布連線狀態
- MQTT 控制連線狀態
- Modbus 採集狀態

## 安全特性
1. **認證閘道**: 通訊服務只有在認證後才啟動
2. **自動清理**: 登出時自動停止服務，避免資源洩漏
3. **狀態監控**: 即時顯示服務狀態，方便故障排除

## 測試步驟

### 1. 啟動系統測試
```bash
cd "c:\Users\A50388.ITRI\Desktop\ScadaEngine"
dotnet run --project ScadaEngine.Web
```

### 2. 登入前檢查
- 開啟瀏覽器至 `https://localhost:5001`
- 應該會自動導向登入頁面
- 此時通訊服務應為未啟動狀態

### 3. 登入後檢查
- 使用帳號 `admin` 密碼 `1234` 登入
- 登入成功後檢查主頁面的通訊服務狀態
- 狀態應該顯示為「已啟動」

### 4. 服務狀態驗證
在主頁面檢查以下狀態：
- [ ] 通訊服務：已啟動
- [ ] MQTT 發布：已連線（需要 MQTT Broker）
- [ ] MQTT 控制：已連線（需要 MQTT Broker）
- [ ] Modbus 採集：採集中（需要 Modbus 設備或模擬器）

### 5. 登出後檢查
- 點擊登出按鈕
- 檢查系統日誌，應該有「通訊服務停止完成」的訊息

## 注意事項
1. **MQTT Broker**: 需要確保 MQTT Broker 運行在 `localhost:1883`
2. **Modbus 設備**: 可以使用模擬器或實體設備
3. **設定檔案**: 確保 `MqttSetting\MqttSetting.json` 和 `Modbus\Modbus.json` 設定正確

## 錯誤處理
- 如果通訊服務啟動失敗，使用者仍可以成功登入
- 錯誤訊息會記錄在系統日誌中
- 主頁面會顯示實際的服務狀態

## 設計優點
1. **安全性**: 確保未經認證的存取無法啟動工業通訊
2. **資源管理**: 自動管理服務生命週期
3. **使用者體驗**: 登入即可使用，登出自動清理
4. **監控性**: 即時顯示服務狀態
5. **錯誤復原**: 服務啟動失敗不影響基本功能

## 技術細節
- 使用依賴注入管理服務
- 線程安全的服務狀態管理
- 異常處理確保系統穩定性
- 符合 MVC 架構的清晰職責分工
