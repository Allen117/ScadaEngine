# 功能說明書 — 能源基準（ISO 50001 能源基線 / EnPI / SEU）

> 頁面：`/EnergyBaseline`（EMS 體系，淡綠主題）
> 對外表述界線：本模組是「以 ISO 50001 / ISO 50006 方法評估能源績效」的軟體落地，**不宣稱**「通過 ISO 50001 驗證」（驗證需政策、稽核、管理審查等制度面，不在軟體範圍）。

## 功能總覽

```
相關變數(X) → 能源基線 EnB(回歸) → 能源績效指標 EnPI(實際/正規化) → 節能量(實際 vs 基線預測)
```

| Tab | 功能 | ISO 50001 對應 |
|---|---|---|
| 基線建模 | 自訂 Y/X 的複線性回歸基線模型，凍結係數 | 相關變數 + 能源基線 EnB |
| EnPI / 節能量報告 | 報告期「實際 vs 基線預測」逐期比較 + Excel 匯出 | EnPI + 節能量（IPMVP 殘差累計） |
| SEU 鑑別 | 迴路用電帕累托排名，累計占比達門檻者標 SEU | 能源審查 6.3 重大能源使用 |

與 EnergyReport / EnergyDeclaration 的差異：既有報表只做「絕對值 + 同期比較」，會被天氣/產量干擾；本模組把能耗對相關變數正規化，扣除外生因素後才是真實績效。

## 基線建模（P1）

### 模型定義
- **目標能耗 Y**：二選一
  - **能源迴路**：走 `EnergyReportService` 計算核心（葉子 boundary 相減 + MaxKwh 溢位 + staleness window + Sign），與用電報表數字一致。
  - **任意點位**：全來源（Modbus / 計算 / DB / OPC UA）點位皆可選。取樣方式二選一：
    - `累計值`：HistoryData 期初期末相減；**值倒退（換錶/重置）該 bucket 直接剔除**（任意點位無 MaxKwh 資訊，不做溢位補正）。
    - `瞬時值`：HistoryData 期間均值（Quality=1）。
    - 選點位後依單位自動推斷（含 "wh" → 累計，其餘 → 均值），可手動覆寫。
- **相關變數 X**：1~**5** 個（上限防過擬合）。每個 X 可為：
  - `點位（均值）`：HistoryData bucket 均值（外氣溫度、濕度、產量…）。
  - `迴路（kWh）`：EnergyReport 計算核心的 bucket kWh。
- **取樣粒度**：`日`（預設，曆日）或 `月`（**曆月**，刻意不走月結期別 — 基線需與電費期界脫鉤，X 變數以自然時間對齊才有物理意義）。
- **基線期**：起訖日（含）。只取「已過完」的 bucket，半天/半月樣本不進回歸。

### 回歸引擎（BaselineRegressionEngine）
- MathNet.Numerics QR 分解求解 OLS（溫濕度共線性下比正規方程式穩定）。
- 輸出：β 係數、截距、R²、調整後 R²、**CV(RMSE)**（= RMSE/mean(Y)，RMSE 用殘差自由度 n−p−1，IPMVP / ASHRAE G14 慣例）、樣本數、各係數 **p-value**（t 分布雙尾；XᵀX 近奇異時顯示「—」）。
- 樣本數 < p+2 拋錯；樣本數 < 變數數×5 或 R² < 0.5 顯示警告（不阻擋）。
- Y 或任一 X 缺值的 bucket 整列剔除，剔除數回報前端。

### 凍結（Freeze）
- 跑完回歸的草稿可「凍結」：`Status=frozen` + `FrozenAt`，之後 EnPI 報告**一律用凍結係數**預測，不重算（基線 = 固定比較基準）。
- 凍結後表單唯讀、不可重算；可「解除凍結」回草稿（前端有警語：既有報告比較基準會改變）。

## EnPI / 節能量報告（P2）

- 選**已凍結**基線 + 報告期（可與基線期不同）→ 逐 bucket：
  - 實際 Y、基線預測 `Ŷ = β0 + Σβi·Xi`（X 用報告期實際值）
  - 節能量 = 預測 − 實際（正值 = 優於基線）
  - 累計節能量（缺值 bucket 不累計，沿用前值）
  - EnPI = 實際 ÷ 預測（< 1 = 優於基線；預測 ≤ 0 顯示「—」）
- 缺 Y 或缺任一 X 的 bucket 標「資料不足」，不入合計。
- 雙軸圖：實際/預測長條（左軸）+ 累計節能量折線（右軸）。
- ClosedXML 匯出（`EnPIReportExcelExporter`，慣例同 EnergyReportExcelExporter；含摘要區 + 逐期明細 + X 變數欄）。

## SEU 重大能源使用鑑別（P3）

- 能源使用單位 = **主要電表的直接子迴路**（同 EMS 用電占比 breakdown 慣例；無主要電表時 fallback 全部根迴路）。
- 指定期間（內部切曆月對齊 chunk 計算，兼顧溢位正確性與查詢量）取各單位 kWh（含子迴路對父層 Sign 補乘）。
- 依 kWh 降冪帕累托排名：占比、累計占比；**累計占比達門檻（預設 80%，可調 1~100）前的項目（含跨越門檻那一項）標為 SEU**。
- kWh ≤ 0 的項目（如回充/量測異常）不入占比分母、不標 SEU，但仍列表。
- 呈現：帕累托圖（SEU 深綠柱 + 累計占比折線）+ 排名表。

## 資料庫

兩表由 `DatabaseSchema.json` 定義，Engine/Web 啟動自動建表（詳見 docs/架構.md §資料庫結構初始化）：

| 表 | 欄位摘要 |
|---|---|
| `EnergyBaseline` | Id, Name, TargetType(point/circuit), TargetSID/TargetCircuitId, TargetMode(cumulative/average), TargetLabel/TargetUnit(快照), Granularity(day/month), BaselineStart/End, Status(draft/frozen), Intercept, R2, AdjR2, CvRmse, SampleCount, FrozenAt, CreatedAt, UpdatedAt, Description |
| `EnergyBaselineVariable` | Id, BaselineId, Sequence(1..5), VarType(point/circuit), SourceSID/SourceCircuitId, Label/Unit(快照), Coefficient, PValue |

點位/迴路名稱以**快照**存於 Label 欄 — 來源改名後模型仍可讀；凍結模型的語意不受來源設定變動影響。

## API（`/EnergyBaseline/api/*`，皆需登入）

| Method | Route | 用途 |
|---|---|---|
| GET | `api/points` | 全來源點位清單（同 /CalcPoint/Points 慣例） |
| GET | `api/circuits` | 迴路平坦清單 |
| GET | `api/models` / `api/models/{id}` | 模型清單 / 單筆（含變數） |
| POST | `api/models` | 新增/更新（id null=新增；更新限草稿並清空舊回歸結果） |
| DELETE | `api/models/{id}` | 刪除（含變數） |
| POST | `api/models/{id}/run` | 執行回歸（限草稿），回統計量+散布圖資料 |
| POST | `api/models/{id}/freeze` / `unfreeze` | 凍結 / 解除凍結 |
| POST | `api/enpi/query` / `api/enpi/export` | EnPI 報告查詢 / Excel 匯出 |
| POST | `api/seu` | SEU 帕累托分析 |

## 注意事項 / 已知邊界

- **月粒度 = 曆月**，與用電報表的「月 = 月結期別」不同 — 屬刻意設計，頁面粒度選項已標示「曆月」。
- 完全共線的變數組合（如同一點位選兩次）QR 仍可能給出數值解，此時 p-value 會是「—」— 判讀請看調整後 R² 與 p-value，勿盲加變數。
- 樣本自由度：月粒度一年僅 12 筆，估 2~3 個係數自由度嚴重不足 — 預設日粒度即為此故。
- 解除凍結會使既有 EnPI 報告的比較基準改變；正式採用後建議凍結不再動，若要調整另建新模型。
