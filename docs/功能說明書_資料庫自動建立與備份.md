# 功能說明書 — 資料庫自動建立與每週備份

## 1. 功能總覽

| 功能 | 執行者 | 時機 | 需要權限 |
|------|--------|------|---------|
| 一次性安裝（建 DB / login / 資料夾 ACL） | `Setting/install-db.ps1` | 安裝時人工執行一次 | 執行者 Windows 帳號為 SQL sysadmin + 本機系統管理員 |
| 自動建 DB 安全網 | Engine / Web 啟動流程 | DB 不存在時 | 連線帳號有 `dbcreator`（無則優雅降級、log 指引） |
| 自動建表 + 補欄位 | Engine / Web 啟動流程（既有功能） | 每次啟動 | 目標 DB `db_owner` |
| 每週自動備份 | Engine `DatabaseBackupService` | 排定時間（預設週日 03:00） | 目標 DB `db_owner` |

設計原則：**安裝時特權與執行時權限分離**。sa 密碼不落地任何設定檔；Engine / Web 日常只用 `dbSetting.json` 的低權限帳號。

## 2. 設定檔 `Setting/DbMaintenanceSetting.json`

```json
{
  "DataFileFolder": "C:\\Scada\\Database",   // 自動建 DB 時 MDF/LDF 位置（不影響既有 DB）
  "BackupFolder": "C:\\Scada\\Backup",       // 備份檔位置
  "BackupEnabled": true,                      // 每週備份總開關
  "BackupDayOfWeek": "Sunday",                // Sunday ~ Saturday
  "BackupTime": "03:00"                       // 24 小時制 HH:mm
}
```

- 檔案不存在或解析失敗 → 使用上述預設值，log warning，不影響啟動
- 部署慣例：`C:\Scada\{Engine, EMS, Database, Backup}` 四資料夾同層
- 改設定後需重啟 Engine 生效（排程於服務啟動時讀取一次）

## 3. 安裝流程

`install-db.ps1` **已綁入部署流程，一般情況不需手動執行**：

| 部署方式 | 觸發點 |
|---------|--------|
| 現場 USB 安裝包 `Install.bat`（BuildRelease.ps1 產出） | 步驟 [4/6]，在還原現場設定**之後**執行（確保讀到現場的 dbSetting.json） |
| 開發機 `QuickDeploy.bat` 選項 1（Install）/ 選項 6（Update） | `DeployService.ps1` 的 Install-Service / Update-Service 內呼叫 `Invoke-DbSetup` |
| 手動 | 以系統管理員執行 `C:\Scada\Engine\App\Setting\install-db.ps1` |

腳本失敗**不會中斷部署**（印出警告與手動重跑指令）— Engine 啟動安全網會兜底建 DB，但備份資料夾 + ACL 只有此腳本會建，失敗時務必補跑。

腳本自動讀同資料夾的 `dbSetting.json` / `DbMaintenanceSetting.json`，依序：

1. 偵測 SQL 服務帳號（預設實例 `NT Service\MSSQLSERVER`、具名實例 `NT Service\MSSQL$實例名`；指定服務名找不到時自動掃描本機唯一 SQL 實例），建立 Database / Backup 資料夾並以 icacls 授予寫入權
2. DB 不存在 → 建立於指定路徑 + `RECOVERY SIMPLE`；已存在 → 跳過，FULL 復原模式時印警告（不自動改）
3. 建立應用 SQL login 並授予目標 DB `db_owner`（已存在則跳過）
4. 建立工程師模式 bootstrap 帳號 `engineer`（詳見 [功能說明書_登入.md](功能說明書_登入.md) §12）：console 互動要求自訂密碼（遮罩、兩次核對、至少 12 碼），或以 `-EngineerPassword` 參數帶入；DB 內已有任何 Engineer 角色帳號（含停用）則跳過 — Update 重跑不會重新長出 bootstrap 帳號。非互動且未帶參數時跳過並警告。Users 表尚未存在（Engine 未曾啟動）時先依 DatabaseSchema.json 相同欄位定義預建（Engine 欄位同步相容）

工程師忘記 `engineer` 密碼時，執行 `reset-engineer-password.ps1`（以 dbSetting.json 應用帳號連線，不需 sysadmin）輸入新密碼直接覆蓋；帳號不存在時自動建立，既有站點補建第一顆 Engineer 帳號也走此腳本。**此工具放在 Release 包根目錄（Install.bat 旁）、刻意不安裝到伺服器** — 由工程師隨包攜帶；BuildRelease/Install.bat/DeployService 都會確保伺服器上不殘留此腳本。

前置需求：執行者 Windows 帳號為 SQL sysadmin（安裝 SQL Server 的帳號預設即是）+ 本機系統管理員。腳本 **idempotent，可重複執行**；參數皆可覆寫（`-ServerInstance "localhost\SQLEXPRESS"` 等），詳見腳本開頭註解。

## 4. 既有環境導入備份

既有站點（DB 已手動建立）跑一次部署流程（QuickDeploy 選項 6 Update 即可）就會自動執行 `install-db.ps1`：偵測到 DB 已存在會跳過建立，但補齊備份資料夾與 SQL 服務帳號權限 — 這是既有環境唯一缺的前置。之後每週備份自動運作，無需其他調整。

## 5. 自動建 DB 安全網（Engine / Web 啟動）

`DatabaseInitializationService.EnsureDatabaseExistsAsync` 掛在建表流程最前，Engine 與 Web 共用：

| 情境 | 行為 |
|------|------|
| DB 存在且可開啟 | 直接通過（既有環境零額外負擔） |
| DB 存在但帳號開不了 | log 錯誤（權限對應問題）+ 修復指引，中止初始化 |
| DB 不存在、帳號有 dbcreator | 自動建立於 `DataFileFolder` + RECOVERY SIMPLE；路徑失敗退 SQL Server 預設路徑 |
| DB 不存在、帳號無權限 | log「請執行 install-db.ps1 或請 DBA 建立（或暫時授予 dbcreator）」，中止初始化、**不 crash** |

降級不造成長期功能缺失：DBA 一次性建好 DB 後，建表 / 補欄位 / 備份全部只需 db_owner。

## 6. 每週自動備份

- **輪替策略**：`{DB名}_A.bak` / `{DB名}_B.bak` 交替 `WITH INIT` 覆寫（優先寫不存在者，兩檔都在則覆寫較舊者）— 恆保留「本次 + 上一次」兩份
- **壓縮**：備份前查 `SERVERPROPERTY('EngineEdition')`，Standard / Enterprise 自動加 `WITH COMPRESSION`，Express 不支援則略過 — 同一套程式兩種環境通用
- **失敗重試**：備份失敗於當日內每小時重試；跨日則等下週排程。Engine 於排定時間停機 → 該週略過，啟動自檢發現最新備份逾 8 天會 log 警告
- **結果可見性**：成功 / 失敗各寫一筆 EventLog（EventType=3 資訊、SID=`_system`；成功 Severity=3 低、失敗 Severity=2 中），於 Web 事件紀錄頁可查；Serilog 同步記錄細節

### 還原方式（災難復原）

```sql
-- 於 SSMS 以 sysadmin 執行；還原會踢掉連線，先停 Engine 與 Web
RESTORE DATABASE [wsnCsharp]
FROM DISK = N'C:\Scada\Backup\wsnCsharp_A.bak'   -- 挑時間較新的那份
WITH REPLACE;
```

## 7. 限制與運維注意

- **備份與 DB 同機同碟只防「DB 損毀」，防不了磁碟故障** — 請將 `C:\Scada\Backup` 納入定期外拷（NAS / 異地）
- 備份檔由 SQL Server 服務進程寫入：資料夾權限是給 SQL 服務帳號，不是 Engine 執行帳號；DB 在遠端主機時 `BackupFolder` 指的是 **DB 那台**的磁碟路徑
- Express 版 DB 上限 10GB，與備份功能無關但為既有限制
- 本功能只做每週完整備份，不做交易記錄備份 — DB 應維持 `RECOVERY SIMPLE`（新建 DB 已自動設定；既有 FULL 模式 DB 由 install-db.ps1 印警告，是否切換由現場 DBA 決定）
