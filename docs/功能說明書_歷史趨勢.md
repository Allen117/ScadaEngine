# 功能說明書：歷史資料 (HistoryData)

## 1. 功能概述

歷史資料頁面提供**多點位時間序列資料查詢與視覺化**能力，使用者可選取 Modbus / 計算 / DB 來源點位，指定時間範圍與取樣間隔後查詢 HistoryData 資料表，以折線圖或柱狀圖呈現歷史數據，並附帶統計摘要與 Excel 匯出功能。

### 核心能力
- **多點位並行查詢**：最多同時查詢 8 個點位，每個點位上限 5000 筆
- **時間間隔取樣**：可選原始 / 1m / 5m / 15m / 30m / 1h，**對整點對齊取每 bucket 首筆**（不平均），便於長區間概覽
- **三種圖表模式**：單點（主圖 + 資料明細表）、雙點（雙 Y 軸）、多點（堆疊小圖）
- **統計分析**：自動計算最小值、最大值、平均值、標準差
- **品質標示**：Bad 品質點位以紅色標記，圖表以 hold-last-good 策略維持曲線連續
- **Excel 匯出**：SheetJS (xlsx) 產生 .xlsx 檔案，含各點位工作表 + 統計摘要表
- **跨頁整合**：接收 ScadaPage 趨勢圖右鍵選單透過 localStorage 傳入的預載點位

## 2. 路由

| 方法 | 路由 | 說明 | 認證 |
|------|------|------|------|
| GET | `/HistoryData` | 歷史資料頁面 | 需登入 |
| GET | `/api/history/data` | 查詢歷史資料 API | 需登入 |

### API 參數與回應

#### GET `/api/history/data`

**查詢參數**：

| 參數 | 型別 | 說明 |
|------|------|------|
| `szSID` | string | 點位 SID（必填） |
| `szStart` | string | 開始時間（ISO 格式，預設 24 小時前） |
| `szEnd` | string | 結束時間（ISO 格式，預設現在） |
| `nInterval` | int | 取樣間隔（分鐘），預設 0。`0` = 原始；`>0` = 對 `Timestamp` 按 N 分鐘對齊整點分 bucket，每 bucket 取最早一筆 |

**回應格式**：

```json
{
  "success": true,
  "szSID": "196865-S1",
  "szName": "溫度感測器",
  "szUnit": "°C",
  "nCount": 1440,
  "isLimited": false,
  "dMin": 22.5,
  "dMax": 28.3,
  "dAvg": 25.12,
  "dStdDev": 1.83,
  "data": [
    { "t": "2026-03-30T08:00:00", "v": 25.3, "q": "Good" },
    { "t": "2026-03-30T08:01:00", "v": 25.1, "q": "Good" }
  ]
}
```

| 欄位 | 說明 |
|------|------|
| `nCount` | 實際資料筆數 |
| `isLimited` | 是否達到 5000 筆上限（true 表示有截斷） |
| `dMin` / `dMax` / `dAvg` | 統計值 |
| `dStdDev` | 標準差（至少 2 筆才計算） |
| `data[].t` | 時間戳記（ISO 格式） |
| `data[].v` | 數值 |
| `data[].q` | 品質（`Good` / `Bad`） |

## 4. 資料流架構

```
使用者開啟 /HistoryData
    │
    ├─ Controller 查詢 DB ──────────────────────────────────
    │   ├─ GetAllCoordinatorsAsync()      → 側欄設備清單
    │   ├─ GetAllModbusPointsAsync()      → 所有 Modbus 點位
    │   └─ GetAllCalculatedPointsAsync()  → 啟用的計算點位（合併入點位清單）
    │
    ├─ Razor SSR ──────────────────────────────────────────
    │   ├─ window.__historyConfig.points       ← JSON 點位陣列
    │   ├─ window.__historyConfig.coordinators ← JSON 設備陣列
    │   └─ window.__historyConfig.initialCoordId ← 首個 Coordinator ID
    │
    ├─ JS 初始化 ──────────────────────────────────────────
    │   ├─ enrichPointsWithDeviceLabel()  ← 為每個點位計算設備名稱前綴
    │   ├─ renderPointList()              ← 依初始 Coordinator 篩選點位
    │   ├─ renderBasket()                 ← 渲染待查詢清單
    │   └─ checkTrendPreload()            ← 檢查 localStorage 預載資料
    │
    └─ 使用者操作 ─────────────────────────────────────────
        ├─ 側欄篩選設備     → renderPointList() 更新點位清單
        ├─ 勾選 + 加入清單  → addToBasket() → renderBasket()
        ├─ 設定時間 + 查詢  → doQuery()
        │   └─ Promise.all(basket.map(fetch /api/history/data))
        │       → buildChart() / buildStackedCharts()
        │       → 更新統計卡片 / 資料明細表
        ├─ 切換圖表類型     → 重新查詢並渲染
        ├─ 切換單軸/雙軸    → 重建圖表
        └─ 匯出 Excel       → exportToExcel()
```

## 5. 控制器說明

### HistoryController

#### `HistoryData()` — 頁面入口

1. 查詢所有 Coordinator（側欄用）+ DB Coordinator
2. 查詢所有 Modbus 點位 + 啟用的計算點位 + DB 來源點位（合併為單一清單）
3. 提取計算點位的群組名稱（供側欄分群）
4. 建構 `CalcGroupMap`（SID → GroupName 對照）
5. 預設時間：過去 24 小時

#### `GetHistoryData()` — 資料查詢 API

1. 驗證 SID 非空、時間範圍合法、`nInterval >= 0`
2. 呼叫 `GetHistoryTableDataAsync(szSID, dtStart, dtEnd, 5000, nInterval)` 查詢 HistoryData
   - `nInterval = 0` 走原 SQL 路徑
   - `nInterval > 0` 走 `ROW_NUMBER() OVER (PARTITION BY bucket ORDER BY Timestamp ASC)` 的 bucket SQL，bucket 公式：`DATEADD(MINUTE, DATEDIFF(MINUTE, 0, Timestamp) / @IntervalMin * @IntervalMin, 0)`
3. 查找點位名稱與單位（依 SID 前綴：Modbus → `CALC-` 計算點位 → `DB` 來源點位）
4. 計算統計值：Min、Max、Avg、StdDev
5. 回傳 JSON（含 `isLimited` 標記是否截斷）

## 7. 前端邏輯 (history.js)

以 IIFE 封裝，對外暴露 `window._history`（`exportToExcel`、`removeFromBasket`）。

### 7.1 初始化

1. 從 `window.__historyConfig` 讀取 Razor 注入的資料
2. `enrichPointsWithDeviceLabel()`：依 SID 數值範圍比對 Coordinator/SubDevice，為每個點位加上 `deviceLabel`（設備名稱）和 `fullName`（`設備/點位名稱`）
3. 檢查 `localStorage['SCADA_TREND_PRELOAD']`，若存在則自動加入 Basket 並刪除

### 7.2 設備側欄

與即時數據頁面相同的三種點擊事件：
- **單一設備** (`coordinator-item`)：篩選該設備下所有點位
- **展開/收合** (`coordinator-toggle`)：展開子設備列表 + 篩選
- **子設備** (`sub-item`)：篩選特定 ModbusID 或計算點位群組

**計算點位特殊處理**：
- `data-id="-999"` 表示計算點位
- 支援群組篩選 (`data-calcgroup`)
- 無群組的計算點位歸類為「未分組」

### 7.3 點位選取與 Basket

**兩階段選取**：

1. **勾選點位**：左側點位清單，checkbox 多選
2. **加入 Basket**：按「加入待查詢清單」按鈕移入

**Basket 規則**：
- 最多 8 個點位 (`MAX_BASKET = 8`)
- 不可重複加入（依 SID 判斷）
- 可跨設備加入
- 每項顯示色彩圓點（對應圖表顏色）
- 可個別移除或全部清除

**色彩常數** (8 色循環)：

| 索引 | 色碼 | 用途 |
|------|------|------|
| 0 | `#0d6efd` (藍) | 第 1 個點位 |
| 1 | `#dc3545` (紅) | 第 2 個點位 |
| 2 | `#198754` (綠) | 第 3 個點位 |
| 3 | `#ffc107` (黃) | 第 4 個點位 |
| 4 | `#0dcaf0` (青) | 第 5 個點位 |
| 5 | `#6f42c1` (紫) | 第 6 個點位 |
| 6 | `#fd7e14` (橘) | 第 7 個點位 |
| 7 | `#6610f2` (靛) | 第 8 個點位 |

### 7.4 查詢流程 (`doQuery`)

1. 驗證：Basket 不為空、時間範圍合法
2. 讀取「時間間隔」下拉值（`#selInterval`，0 = 原始）
3. 顯示 Loading spinner
4. `Promise.all`：並行呼叫 `/api/history/data?...&nInterval=N` 查詢每個 Basket 中的點位
4. 處理回應：
   - 任一失敗 → 顯示錯誤訊息
   - 全部無資料 → 顯示「該時段內無歷史資料」
   - 有資料被截斷 → 顯示黃色警告 Badge
5. 根據有效點位數量決定顯示模式（見 7.5）
6. 更新統計卡片 / 統計表格
7. 顯示匯出 Excel 按鈕
8. 單點模式額外顯示資料明細表

### 7.5 圖表渲染模式

| 有效點位數 | 圖表模式 | 特殊功能 |
|-----------|----------|---------|
| **1 個** | 單一主圖（380px） | 統計卡片（Min/Max/Avg/StdDev）、資料明細表、品質紅色標點、面積填充 |
| **2 個** | 雙 Y 軸主圖 | 左右軸分別對應不同點位，可切換單軸/雙軸模式 |
| **≥3 個** | 堆疊小圖（每個 170px） | 每個點位獨立 Y 軸，共用 X 時間軸（僅最後一個小圖顯示 X 刻度） |

### 7.6 Chart.js 配置

**時間軸**：
- 使用 `date-fns` adapter
- 顯示格式：秒 → `HH:mm:ss`、分 → `MM-dd HH:mm`、時 → `MM-dd HH:mm`、日 → `MM-dd`
- Tooltip 格式：`yyyy-MM-dd HH:mm:ss`

**品質處理 (`holdLastGoodValue`)**：
- Bad 品質的資料點在圖表上使用前一個 Good 值，保持曲線連續
- 原始值保留供 Tooltip 顯示
- 單點模式下 Bad 品質點以紅色標記

**效能最佳化**：
- 資料量 > 500 筆：隱藏圓點 (`pointRadius: 0`)
- 資料量 > 1000 筆：停用動畫 (`animation.duration: 0`)

**雙 Y 軸**：
- 左軸 (`y`)：第 1 個點位，軸標題顯示單位
- 右軸 (`y1`)：第 2 個點位，不繪製網格線
- 各軸自動計算 ±10% padding 的量程範圍

### 7.7 單軸/雙軸切換

僅在 **2 個有效點位** 時顯示切換按鈕：
- **雙軸模式**（預設）：兩個點位各自 Y 軸，量程獨立
- **單軸模式**：兩個點位共用左側 Y 軸
- 切換時以 `lastResults` 快取重建圖表，無需重新查詢 API

### 7.8 圖表類型切換

- **折線圖** (`line`)：預設，單點模式含面積填充
- **柱狀圖** (`bar`)
- 切換時觸發 `doQuery()` 重新查詢與渲染

## 11. 快速時間範圍

查詢條件區提供快速設定按鈕：

| 按鈕 | data-hours | 效果 |
|------|-----------|------|
| 1h | 1 | 最近 1 小時 |
| 6h | 6 | 最近 6 小時 |
| 24h | 24 | 最近 24 小時 |
| 3天 | 72 | 最近 3 天 |
| 7天 | 168 | 最近 7 天 |

點擊後自動填入開始/結束時間（結束時間為當下）。

### 時間間隔下拉 (`#selInterval`)

位於時間範圍區域結束時間下方。選項：

| value | 顯示 | 行為 |
|-------|------|------|
| `0` | 原始（預設）| 不取樣，回傳區間內所有 raw 紀錄（上限 5000） |
| `1` | 1m | 每分鐘整點分 bucket，取首筆 |
| `5` | 5m | 每 5 分鐘對齊（`HH:00 / HH:05 / HH:10 / ...`） |
| `15` | 15m | 每 15 分鐘對齊 |
| `30` | 30m | 每 30 分鐘對齊 |
| `60` | 1h | 整點對齊 |

切換間隔後**不會自動重查**，需按「查詢」按鈕生效。

## 12. 跨頁整合

### ScadaPage → HistoryData 趨勢圖預載

**寫入端**（ScadaPage）：
```javascript
localStorage.setItem('SCADA_TREND_PRELOAD', JSON.stringify({
    sids: [{ sid: "196865-S1", name: "溫度", unit: "°C" }]
}));
```

**讀取端**（History）：
1. `DOMContentLoaded` 時檢查 `localStorage['SCADA_TREND_PRELOAD']`
2. 若存在，解析並自動加入 Basket
3. 從 `allPoints` 補充完整名稱（`fullName`）與單位
4. 讀取後立即刪除 localStorage key
