# 功能說明書 — 氣象資料來源（/WeatherSetting）

把中央氣象署（CWA）開放資料的**外氣溫度、外氣相對濕度**自動導入 SCADA pipeline，
成為一般 DB 來源點位，下游（即時數據、歷史趨勢、警報、**能源基線相關變數 X**、EMS）全部通用免改。

## 資料流

```
中央氣象署開放資料 API（opendata.cwa.gov.tw）
    ↓ HTTPS GET /api/v1/rest/datastore/{DatasetId}?Authorization={key}&StationId={id}（逾時 15s）
WeatherFetchService（Web BackgroundService，每 30 秒檢查設定、依間隔到期抓取）
    ↓ UPDATE DBLatestData（以 DBCoordinator.Name='Weather' + DBPoints.Sequence JOIN 定位，
    │  同 UpdateTimeSim.sql 慣例 — CoordinatorId 是 IDENTITY 換環境會變，不寫死 SID）
DBLatestData（Weather S1=外氣溫度、S2=外氣相對濕度、S3=外氣濕球溫度〔由 S1/S2 推導〕；
    │  Timestamp=CWA 觀測時間）
    ↓ Engine DbCommunicationService polling（既有 DB 來源 pipeline，見 docs/架構.md §DB 來源 Flow）
HistoryData / LatestData / MQTT → 警報 / 計算點 / 能源基線 / Web 即時數據
```

- **點位定義**：`ScadaEngine.Engine/DBPoint/Weather.json`（S1 外氣溫度 °C、S2 外氣相對濕度 %、S3 外氣濕球溫度 °C）。
  Sequence 由陣列順序決定，**永不插隊** — SID 被 HistoryData / 基線引用。
- **S3 濕球溫度為系統推導點位**（非 CWA 欄位）：`WeatherSettingService.WriteObservationAsync` 於每次寫入
  S1/S2 時以 Stull (2011) 經驗式（`ScadaEngine.Common/Algorithms/Psychrometrics.WetBulbStull`）從溫濕度算出，
  與 S1/S2 同觀測時間戳、同 Quality 語意。適用域 T 0~50°C、RH 5~100%（實測 0°C 以下 Stull 擬合劣化故收斂下限），
  範圍外回 NaN → 走壞值路徑 Quality=0 保留最近成功值。
  **既有部署升級注意**：Weather.json 加點後需重啟 Engine（或發 `SCADA/Sys/DbCoordinator/Reload` MQTT）
  才會 seed DBPoints/DBLatestData 的 S3 列；在此之前 Web 寫 S3 為 0 列（無害，S1/S2 照常）。
- **抓取者在 Web 而非 Engine / 外部排程**：測站選單需要 UI 與設定儲存（DB 表），Web 直讀設定改完即生效
  （最慢 30 秒），且客戶部署零額外步驟（不需逐台註冊 Task Scheduler 工作）。

## 設定頁（/WeatherSetting，系統選單 →「氣象資料」）

1. **API 授權碼**：至 https://opendata.cwa.gov.tw 免費註冊 → 取得 `CWA-…` 授權碼填入。
   **每個部署現場用自己的 key**，勿多現場共用 — 流量配額以授權碼計、共用金鑰重產全體斷線、濫用無法歸屬。
2. **載入測站清單**：用當下輸入的 key 呼叫 CWA（Web 後端代理，key 不暴露於瀏覽器直連），
   合併兩個資料集、只列**溫度與濕度當下皆有效**的測站，快取 1 小時：
   - `O-A0001-001` 自動氣象站（站多、涵蓋全台鄉鎮）
   - `O-A0003-001` 署屬有人站（主要城市，品質高）
3. **縣市 → 測站**兩層下拉選站；**測試連線**立即抓一次並顯示讀到的溫濕度。
4. **抓取間隔** 5–1440 分鐘（預設 10；CWA 觀測約每 10 分鐘更新，抓更密沒意義）＋**啟用**開關 → 儲存。
5. **執行狀態**卡顯示最近抓取時間/結果/訊息（每 30 秒自動刷新）。

換測站只是換資料來源，SID 不變，下游引用不斷。

## Quality 規則（防「舊溫度當好值餵基線」）

| 情境 | 行為 |
|------|------|
| 抓取成功、觀測時間距今 ≤ 60 分 | 寫 Value + Timestamp（=CWA 觀測時間）+ Quality=1 |
| 個別欄位缺測（CWA 哨兵值 -99） | 該點位只降 Quality=0（保留最近成功值），另一點位照常寫；S3 因輸入不全一併 Quality=0 |
| 觀測時間距今 > 60 分（測站掛掉 API 仍回舊資料） | 三點位 Quality=0 |
| API 失敗（網路/授權/逾時）、查無測站 | 三點位 Quality=0 |
| 溫濕度超出濕球公式適用域（T 0~50°C、RH 5~100%） | 僅 S3 Quality=0，S1/S2 照常寫好值 |
| 啟用 → 停用的瞬間 | 補寫一次 Quality=0，讓下游知道資料已失效 |

CWA 回應解析要點（`WeatherCwaParser`，2026-07-15 對真 API 驗證）：數值欄位皆為字串、
缺測哨兵 `-99`（含 `-99.0`，以 ≤ -90 判定）、觀測時間在 `ObsTime.DateTime`（ISO 8601 含 +08:00）。

## LastFetchMessage 機器碼

DB 存機器碼、設定頁前端依語系翻譯（同警報 MessageKey 精神的輕量版）：

| 碼 | 意義 |
|----|------|
| `ok\|{temp}\|{hum}\|{time}` | 成功 |
| `missing\|{temp}\|{hum}\|{time}` | 部分欄位缺測（`-` 表缺） |
| `stale\|{time}` | 觀測過舊 |
| `station_not_found` | 查無所選測站（可能裁撤） |
| `coordinator_not_loaded` | Engine 尚未載入 Weather.json（DBLatestData 無 seed 列） |
| `disabled` | 已停用 |
| `error\|{detail}` | 例外（HTTP / 授權 / 逾時…） |

## 資料表

`WeatherSetting`（單列 Id=1，DatabaseSchema.json 啟動自建）：
ApiKey / DatasetId / StationId / StationName / County（測站快照）/ PollIntervalMinutes / IsEnabled
/ LastFetchTime / LastFetchOk / LastFetchMessage / UpdatedAt / CreatedAt。

## 部署注意

- **Weather.json 需部署到 Engine 執行目錄**（`DBPoint/`）並重啟 Engine 或發 `SCADA/Sys/DbCoordinator/Reload`，
  Engine 才會建 Coordinator/Points 並 seed DBLatestData；未載入前抓取會回報 `coordinator_not_loaded`。
- **本機需可對外 HTTPS 連 `opendata.cwa.gov.tw`**（防火牆白名單）。
- 出廠 ApiKey 為空 = 功能停用；開發測試 key 不得寫死進程式碼或預設值。
