# 功能說明書 — 國定假日設定

> 路由：`/HolidaySetting`（EMS 體系，能管設定選單下）
> 12 個月年曆點選日期標註國定假日/離峰日；標註日在時間電價（TOU）計價時以 `sun_offday`（週日及離峰日）費率落段。
> 搭配電費計算使用，見 [功能說明書_電費設定.md](功能說明書_電費設定.md) §電費計算。

---

## 功能總覽

- **整年 12 個月年曆**（RWD 格線：4/3/2/1 欄自適應），左右箭頭切換年份
- **點選日期切換標註/取消**；標註日紅底高亮、週六/週日淡黃底視覺區別（週末本來即非平日費率，不需標註）、今日綠框
- **整年批次覆蓋儲存**：前端點選只改工作副本，按「儲存」一次送出該年全部標註日 → 後端「刪該年 + 重插」，天然冪等
- 未儲存變更提示（工具列警示 + beforeunload 攔截 + 切換年份 confirm）
- zh-TW / en 雙語；EMS 綠色主題自動套用；權限管理（AccountSetting）可單獨授權

## 計價語意（DayType 判定）

TOU 落段時 DayType 判定順序 — **假日優先於星期**：

1. 日期在 `Holidays` 表 → `sun_offday`（週日及離峰日）
2. 週六 → `sat`
3. 週日 → `sun_offday`
4. 其餘 → `weekday`

- 對齊台電「週日及離峰日」定義；補班日（週六上班）若需按平日計價，**取消標註即可**（未標註的週六本來就是 `sat`，補班日台電仍按週六計價，一般不需處理）
- **編修假日只影響之後的自動計價**；已算過的歷史區間不自動重算 — 儲存成功訊息提示「如需回溯請至電費設定執行重新計算」（避免隱性大量重算）
- flat / progressive 方案不分日別，標註與否不影響

## 資料儲存

`Holidays` 表（`DatabaseSchema.json` 定義，Engine/Web 啟動自動建表）：

| 欄位 | 型別 | 說明 |
|---|---|---|
| HolidayDate | date PK | 標註日期（一列一日，窄表） |
| CreatedAt | datetime default getdate() | |

`HolidayService` 全表 static 快取（跨 Scoped 實例共用，寫入時失效）— 計價端逐時查 DayType 不打 DB。

## API

| Method | 路徑 | 說明 |
|---|---|---|
| GET | `/HolidaySetting/api/holidays?year=` | 該年度所有標註日期（`{ year, dates: ["yyyy-MM-dd", ...] }`） |
| POST | `/HolidaySetting/api/holidays` | 整年批次覆蓋 `{ year, dates: [...] }`（未列入者視為取消；僅接受該年度日期） |

## 已知限制

- 年份範圍防呆 2000–2100
- 無「匯入台電行事曆」功能 — 假日需逐年手動標註
- 標註不分「假日類型」（國定假日/彈性放假/離峰日一律同一種標註）
