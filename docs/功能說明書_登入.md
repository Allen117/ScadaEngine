# 登入功能（Login）說明書

> 文件版本：1.0　｜　產出日期：2026-03-31　｜　對應路徑：`ScadaEngine.Web/Features/Login/`

---

## 1. 功能概述

登入功能是 SCADA Web 系統的入口閘門，負責使用者身分驗證與 Session 管理。系統採用 **Cookie-based 認證**，搭配角色權限控制（Admin / User），確保只有授權人員能操作監控系統。

---

## 2. 頁面路由

| 方法 | 路由 | 授權 | 說明 |
|------|------|------|------|
| GET | `/Login` | AllowAnonymous | 顯示登入表單 |
| POST | `/Login` | AllowAnonymous | 提交帳密驗證 |
| POST | `/Login/Logout` | Authorize | 登出（需 AntiForgeryToken） |
| GET | `/Login/AccessDenied` | AllowAnonymous | 無權限提示頁 |
| GET | `/` | — | 根路徑，重導至 `/Login` |

## 4. 認證流程

### 4.1 登入流程圖

```
使用者開啟 /Login
       │
       ▼
  已登入？ ──是──→ 重導 /ScadaPage
       │
      否
       │
       ▼
  顯示登入表單
  （DB 無 Admin 時顯示預設帳密提示）
       │
  使用者填寫帳密 → POST /Login
       │
       ▼
  ① ModelState 驗證
     └─ 失敗 → 顯示錯誤訊息
       │
      通過
       │
       ▼
  ② DB 帳號驗證（SHA256 密碼比對）
     └─ 成功 → 載入角色 + 權限 → 建立 Cookie → 重導 /ScadaPage
       │
      失敗
       │
       ▼
  ③ DB 無 Admin？嘗試預設帳密（admin / admin）
     └─ 成功 → 以 Admin 角色建立 Cookie → 重導 /ScadaPage
     └─ 失敗 → 顯示「使用者名稱或密碼錯誤」
```

### 4.2 密碼驗證機制

| 驗證方式 | 觸發條件 | 比對邏輯 |
|---------|---------|---------|
| DB 驗證 | Users 表有資料 | `SHA256(輸入密碼).ToLower()` == `PasswordHash` 欄位 |
| 預設帳密 | Users 表無任何啟用的 Admin | 帳號 `admin`（不分大小寫）、密碼 `admin`（區分大小寫） |

**安全設計**：
- 驗證失敗後立即呼叫 `ClearSensitiveData()` 清除記憶體中的密碼
- 密碼欄位標記 `[DataType(DataType.Password)]`，不會回填至表單
- POST 表單包含 `AntiForgeryToken` 防止 CSRF 攻擊

## 5. Cookie 認證配置

### 5.1 Cookie 設定（Program.cs）

| 參數 | 值 | 說明 |
|------|-----|------|
| Cookie 名稱 | `ScadaAuth` | 瀏覽器儲存的 Cookie 名稱 |
| LoginPath | `/Login/Index` | 未登入時重導路徑 |
| LogoutPath | `/Login/Logout` | 登出路徑 |
| AccessDeniedPath | `/Login/AccessDenied` | 無權限重導路徑 |
| 預設過期 | 4 小時 | 一般登入 |
| 記住我過期 | 30 天 | 勾選「記住我」 |
| SlidingExpiration | true | 持續使用時自動延長 |
| HttpOnly | true | JavaScript 無法讀取 Cookie |
| SecurePolicy | SameAsRequest | HTTP/HTTPS 皆可 |

### 5.2 認證 Claims

登入成功後寫入 Cookie 的宣告資訊：

| Claim | 內容 | 用途 |
|-------|------|------|
| `ClaimTypes.Name` | 使用者帳號 | 識別身分 |
| `ClaimTypes.NameIdentifier` | 使用者帳號 | 識別身分（重複） |
| `ClaimTypes.Role` | `"Admin"` 或 `"User"` | 角色判斷 |
| `Permissions` | 權限 JSON 字串 | User 角色的頁面權限 |
| `LoginTime` | `yyyy-MM-dd HH:mm:ss` | 登入時間戳記 |

### 5.3 權限 JSON 結構（User 角色）

```json
{
  "pages": ["/ScadaPage", "/RealTime", "/History/Trend"],
  "scadaPages": {
    "p1": { "canView": true, "canControl": false },
    "p2": { "canView": true, "canControl": true }
  }
}
```

- `pages`：可存取的主要頁面路由清單
- `scadaPages`：SCADA 監控頁的細部權限（按頁面 SID 分配檢視/控制權限）
- Admin 角色不需要權限 JSON（擁有全部權限）

## 6. 登出機制

### 6.1 登出流程

```
使用者點擊 Navbar「登出」
       │
       ▼
  顯示確認 Modal（「確定要登出嗎？」）
       │
  確認 → 提交隱藏 form（POST /Login/Logout + AntiForgeryToken）
       │
       ▼
  HttpContext.SignOutAsync() → 清除 Cookie → 重導 /Login
```

### 6.2 實作細節

- **Navbar** 的登出按鈕觸發 `showLogoutModal()` 顯示 Bootstrap Modal
- 確認後呼叫 `confirmLogout()` 提交隱藏的 `<form id="logoutForm">`
- 隱藏 form 使用 `POST` + `@Html.AntiForgeryToken()`
- **禁止使用 GET 連結登出**（GET 無法攜帶 AntiForgeryToken，違反安全規範）

---

## 7. 無權限存取頁面（AccessDenied）

### 7.1 觸發時機

- User 角色嘗試存取未授權的頁面時（由 `PageAccessFilter` 全域過濾器檢查）
- Cookie 認證中介軟體偵測到角色不符時

### 7.2 頁面內容

- 顯示 403 圖示 + 「無權限存取」提示
- 提供「回到首頁」按鈕（導向 `/ScadaPage`）
- 使用主要 Layout（含 Navbar），非獨立頁面

## 8. 資料模型

### 8.1 LoginModel

**位置**：`Features/Login/Models/LoginModel.cs`

| 屬性 | 型別 | 驗證 | 說明 |
|------|------|------|------|
| `szUserName` | string | `[Required]` | 使用者名稱 |
| `szPassword` | string | `[Required]` | 密碼（不回填） |
| `isRememberMe` | bool | — | 記住我選項（預設 false） |

**方法**：

| 方法 | 回傳 | 說明 |
|------|------|------|
| `IsInputValid()` | bool | 檢查帳號密碼皆非空白 |
| `ClearSensitiveData()` | void | 清除密碼欄位（安全防護） |

### 8.2 相關資料庫操作

| Repository 方法 | SQL 邏輯 | 用途 |
|----------------|---------|------|
| `GetAdminCountAsync()` | `SELECT COUNT(*) FROM Users WHERE Role='Admin' AND IsActive=1` | 判斷是否顯示預設帳密提示 |
| `ValidateUserAsync(user, pass)` | 比對 `SHA256(pass).ToLower()` 與 `PasswordHash` | DB 密碼驗證 |
| `GetUserByUsernameAsync(user)` | 查詢 Users 表取得完整使用者資料 | 取得角色資訊 |
| `GetUserPermissionsAsync(userId)` | 查詢 UserPermissions 表 | 載入 User 角色權限 JSON |
| `UpdateLastLoginAsync(user)` | `UPDATE Users SET LastLoginAt=GETDATE()` | 記錄最後登入時間 |

## 9. 頁面 UI

### 9.1 登入頁面（Index.cshtml）

- **Layout**：使用獨立的 `_LoginLayout.cshtml`（不含 Navbar/Sidebar）
- **表單元素**：
  - 使用者名稱輸入框（`<input>` + `autocomplete="username"`）
  - 密碼輸入框（`type="password"` + `autocomplete="current-password"`）
  - 記住我核取方塊
  - 登入按鈕（全寬 `btn-primary btn-lg`）
- **預設帳密提示**：僅在 DB 無啟用 Admin 時顯示（`ViewBag.ShowDefaultHint`）
- **錯誤訊息**：以 `alert alert-danger` 區塊顯示驗證失敗訊息
- **防護**：表單包含 `@Html.AntiForgeryToken()`

### 9.2 樣式設計

| 元素 | 樣式 |
|------|------|
| 登入標題 | `#2c5aa0` 配合盾牌圖示 |
| 登入按鈕 | 漸層 `linear-gradient(135deg, #4a90e2, #357abd)` |
| 輸入框 Focus | 光暈 `rgba(74, 144, 226, 0.25)` |
| 卡片 | 居中置中，`shadow-sm` 陰影 |

### 9.3 無權限頁面（AccessDenied.cshtml）

- **Layout**：使用主要 `_Layout.cshtml`（含 Navbar）
- 垂直置中顯示 `fa-ban` 紅色圖示
- 文字提示 + 回到首頁按鈕

## 10. 安全考量

| 項目 | 實作方式 |
|------|---------|
| CSRF 防護 | `[ValidateAntiForgeryToken]` + `@Html.AntiForgeryToken()` |
| 密碼儲存 | SHA256 雜湊（非明文），DB 只存 Hash |
| 密碼傳輸 | HTTPS（生產環境）/ HTTP（開發環境 :5038） |
| Cookie 安全 | HttpOnly=true（防 XSS 讀取）、SlidingExpiration |
| 敏感資料清除 | 驗證後立即 `ClearSensitiveData()` |
| 登入失敗日誌 | `LogWarning` 記錄失敗的使用者名稱 |
| 預設帳密限制 | 僅在 DB 無 Admin 帳號時生效，建立帳號後自動失效 |
