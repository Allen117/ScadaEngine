# 功能說明書：需量計算服務（DemandCalculatorService）

**版本**：1.0.0
**日期**：2026-06-29
**模組**：ScadaEngine.Engine

---

## 一、功能概述

台電以 **15 分鐘固定窗口**計費需量（契約需量）。本功能在 Engine 端每分鐘計算一次「過去 15 分鐘的時間加權平均功率（kW）」（滾動窗口），讓值班人員即時掌握各電表迴路的當前需量趨勢，在接近契約需量時提早採取措施。

> ⚠️ 滾動窗口與台電固定窗口不對齊，無法保證不超約。本功能為監控/預警工具，**非計費工具**。

---

## 二、資料流

```
EnergyCircuit.DemandSID (DB 設定)
        ↓ 啟動時載入
DemandCalculatorService
        ↓ 每分鐘
HistoryData 表 (15min 窗口)  →  TWA 計算  →  DemandData 表
```

---

## 三、啟用條件

- `EnergyCircuit` 表中的葉子節點 `DemandSID` 欄位**不為 NULL** 時，該迴路啟用需量計算。
- `DemandSID` 填寫對應功率點位的 SID（例如 Modbus 功率點 `196865-S2`、計算點 `CALC-S3`）。
- 同一個 SID 即使被多個 EnergyCircuit 行共用，也只計算一次（`DISTINCT` 去重）。

---

## 四、計算邏輯

### 4.1 執行頻率

- Engine 啟動後，先對齊到下一個整分鐘（xx:00:00）再開始計算，避免第一次計算到不完整的窗口。
- 之後每分鐘觸發一次（計算時刻 `T = DateTime.Now` 截到整分鐘）。

### 4.2 窗口定義

| 欄位 | 說明 |
|------|------|
| WindowStart | `T - 15 分鐘` |
| WindowEnd (= T) | 計算時刻（整分鐘） |

查詢 `HistoryData` 中 `Timestamp BETWEEN WindowStart AND T` 且 `Quality = 1` 的樣本，並依 Timestamp 升冪排序。

### 4.3 時間加權平均（TWA，step-hold 模型）

```
totalWeighted = 0, totalSeconds = 0
for i = 0 .. N-2:
    dt = points[i+1].Timestamp - points[i].Timestamp  (秒)
    totalWeighted += points[i].Value × dt
    totalSeconds  += dt

// 最後一段延伸到 windowEnd（T）
lastDt = T - points[N-1].Timestamp  (秒)
totalWeighted += points[N-1].Value × lastDt
totalSeconds  += lastDt

DemandKW = totalWeighted / totalSeconds
```

> 採 step-hold 模型的原因：Modbus 掉線補點、Engine 重啟等情況點距不均，若用簡單平均（SUM/COUNT）會使短點距資料過度影響結果。

### 4.4 資料品質判斷

| 條件 | DemandKW | Quality |
|------|----------|---------|
| Quality=1 樣本數 ≥ 5 | 時間加權平均值 | 1（Good） |
| Quality=1 樣本數 < 5 | 0 | 0（資料不足） |

## 五、資料庫設計

### DemandData 表

| 欄位 | 型別 | PK | 說明 |
|------|------|----|------|
| SID | nvarchar(100) | ✅ | 對應 EnergyCircuit.DemandSID |
| Timestamp | datetime | ✅ | 計算時刻（truncate 到整分鐘，local time） |
| DemandKW | float | | 15min 滑動 TWA 功率（kW）；Quality=0 時為 0 |
| WindowStart | datetime | | 窗口起點（= Timestamp - 15min） |
| SampleCount | int | | 窗口內 Quality=1 樣本數 |
| Quality | tinyint | | 1=Good, 0=資料不足（SampleCount < 5） |

- Clustered Index: (SID, Timestamp)
- 採 MERGE 寫入（同一分鐘重算可覆寫，不產生重複）

---

## 六、熱重載

`DemandCalculatorService` 提供公開方法 `ReloadDemandSidsAsync()`。

當 Web 端修改 `EnergyCircuit.DemandSID` 設定後，可透過 MQTT Reload Subscriber 呼叫此方法，讓 Engine 立即重新讀取最新的 DemandSID 清單，無需重啟服務。

> 目前 Web 端尚未發布 EnergyCircuit Reload MQTT 信號，`ReloadDemandSidsAsync()` 供未來擴充使用。

---

## 七、IDataRepository 新增方法

| 方法 | 說明 |
|------|------|
| `GetDemandSidsAsync()` | 取得 EnergyCircuit 中所有 DISTINCT DemandSID（NULL 排除） |
| `UpsertDemandDataAsync(model)` | UPSERT 一筆計算結果至 DemandData 表 |

---

## 八、已知限制

- 此功能只負責 Engine 端計算與落庫，**無 Web UI**。Web 顯示（如即時需量儀表）另立 plan。
- 15 分鐘滾動窗口與台電計費窗口（每整點 0/15/30/45 分）**不對齊**。
- 不加自動清理，DemandData 全量保留；未來依需求另行規劃清理策略。
