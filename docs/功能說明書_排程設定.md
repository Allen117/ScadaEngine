# 功能說明書：排程設定 (ScheduleSetting)

## 1. 功能概述

排程設定頁面提供**時間排程的 CRUD 管理介面**。建立的排程記錄存於 `TimeSchedules` 資料表，由 Engine 端 `LogicFlowExecutionService` 透過「A 接點 — 排程模式」讀取使用：當系統時間落在排程的有效時段時，A 接點自動導通；否則斷開。Engine 每 15 秒會 reload 一次排程快取。

### 核心能力
- **四種重複模式**：每週、N 週循環、每月、N 月循環
- **跨日時段**：例如夜班 22:00 ~ 01:00 用單筆排程設定即可，不需切兩筆
- **例外日**：當天整段不啟動（國定假日、停機日、設備檢修日）
- **加開日**：當天無視重複規則強制啟動（補班日、活動日）
- **自動防呆**：同日期不可同時出現在例外日與加開日，前後端都驗證
- **即時狀態顯示**：每筆排程在表格中顯示「導通中／等待中／非排程日／停用」

## 3. 資料表 `TimeSchedules`

| 欄位 | 型態 | 必填 | 說明 |
|------|------|------|------|
| `Id` | int (IDENTITY, PK) | ✓ | 流水號 |
| `Name` | nvarchar(100) | ✓ | 排程名稱，例如「上班時間」 |
| `RecurrenceType` | tinyint | ✓ | 0=每週 1=N 週循環 2=每月 3=N 月循環 |
| `RunLength` | int | type 1,3 | 跑幾週/月，例如 1=跑 1 週 |
| `RestLength` | int | type 1,3 | 休幾週/月，例如 1=休 1 週 |
| `AnchorDateTime` | datetime | type 1,3 | 週期起算時刻（決定跑/休交替切換點） |
| `DaysOfWeek` | nvarchar(50) | type 0,1 | 星期幾，逗號分隔 ISO 1=週一..7=週日，例如 `1,2,3,4,5` |
| `DaysOfMonth` | nvarchar(100) | type 2,3 | 每月幾號，逗號分隔，例如 `1,15` |
| `StartTime` | nvarchar(10) | ✓ | HH:mm，例如 `08:00` |
| `EndTime` | nvarchar(10) | ✓ | HH:mm，例如 `17:00`；若 ≤ StartTime 則為跨日 |
| `ExcludeDates` | nvarchar(MAX) | | 例外日清單，逗號分隔 yyyy-MM-dd |
| `IncludeDates` | nvarchar(MAX) | | 加開日清單，逗號分隔 yyyy-MM-dd |
| `IsEnabled` | bit | ✓ | 1=啟用 0=停用 |
| `Remarks` | nvarchar(200) | | 備註 |
| `CreatedAt` / `UpdatedAt` | datetime | | 系統維護 |

## 4. 判斷邏輯（Engine 端 `EvalScheduleIsActive`）

每秒對每筆啟用中的排程做一次評估，回傳「目前是否導通」。

### 4.1 流程

```
1. 取得 startMin / endMin（轉成分鐘數）
2. 判斷 isCrossDay = (endMin <= startMin)
3. 決定 baseDate（日期條件比對基準）：
     非跨日：startMin <= nowMin < endMin → baseDate = 今天
              否則 → return false
     跨日：  nowMin >= startMin → baseDate = 今天（晚上段）
              nowMin <  endMin   → baseDate = 昨天（凌晨段，屬於昨天那筆排程的延續）
              否則（中午段）     → return false
4. CheckDayMatch(record, baseDate)：
     a. baseDate ∈ IncludeDates → return true   (加開日無視規則)
     b. baseDate ∈ ExcludeDates → return false  (例外日整天停)
     c. 依 RecurrenceType switch：
        0 → CheckDaysOfWeek(baseDate)
        1 → CheckWeekCycle(baseDate) && CheckDaysOfWeek(baseDate)
        2 → CheckDaysOfMonth(baseDate)
        3 → CheckMonthCycle(baseDate) && CheckDaysOfMonth(baseDate)
```

### 4.2 跨日範例

設定「每週一 22:00 ~ 01:00」：

| 現在時刻 | nowMin vs startMin/endMin | baseDate | 預期 |
|------|------|------|------|
| 週一 21:30 | nowMin < endMin（false，因為現在 < 22:00 也 < 01:00 失效） | 中午段 | OFF |
| 週一 22:30 | nowMin >= startMin | 週一 | CheckDaysOfWeek(週一) = true → ON |
| 週二 00:30 | nowMin < endMin | 週一（昨天） | CheckDaysOfWeek(週一) = true → ON |
| 週二 01:30 | nowMin > endMin、nowMin < startMin | 中午段 | OFF |
| 週二 22:30 | nowMin >= startMin | 週二 | CheckDaysOfWeek(週二) = false → OFF |

### 4.3 例外日／加開日範例

設定「每週一 09:00 ~ 17:00」、加例外日 `2026-05-11`：

| 日期 | 時刻 | 結果 |
|------|------|------|
| 週一 2026-05-04 | 10:00 | ON |
| 週一 2026-05-11 | 10:00 | OFF（例外日生效） |
| 週一 2026-05-18 | 10:00 | ON |

設定「每週一 09:00 ~ 17:00」、加加開日 `2026-05-13`：

| 日期 | 時刻 | 結果 |
|------|------|------|
| 週三 2026-05-13 | 10:00 | ON（加開日強制啟動，無視「每週一」規則） |
| 週四 2026-05-14 | 10:00 | OFF（無加開、規則不符） |

跨日 + 例外日：「每週一 22:00 ~ 01:00」、加例外 `2026-05-04`（週一）：

| 日期 | 時刻 | baseDate | 結果 |
|------|------|------|------|
| 週一 2026-05-04 | 22:30 | 2026-05-04 | OFF（例外日） |
| 週二 2026-05-05 | 00:30 | 2026-05-04（昨天） | OFF（例外日延伸到凌晨段） |

## 5. 防呆規則

### 5.1 例外日 vs 加開日交集

- **前端**：將日期加到 A 清單時若已在 B 清單，跳 `confirm()`：「此日期已在 B，是否改為 A？（會自動從 B 移除）」確認則自動轉移。
- **後端 Controller**：`Save` Action 執行 `intersect` 檢查，若交集非空回 400，訊息為「日期 yyyy-MM-dd 同時出現在例外日與加開日，請擇一」。
- **後端格式驗證**：每個日期須為合法 `yyyy-MM-dd`，不合法回 400。輸出時自動去重 + 排序。

### 5.2 其他驗證
- 排程名稱不可空
- StartTime / EndTime 不可空、不可相同
- type 0,1 → DaysOfWeek 至少選一天
- type 2,3 → DaysOfMonth 至少選一天
- type 1,3 → RunLength ≥ 1、RestLength ≥ 1、AnchorDateTime 必填

## 6. 列表 UI

「日期條件」欄位若有例外/加開資料，會在原本的星期/日期 badge 下方加灰字提示，例如：

```
週一 週二 週三 週四 週五
（例外 2 日｜加開 1 日）
```

詳細日期內容在編輯 Modal 中以彩色 chip（紅色＝例外、綠色＝加開）顯示，可一鍵移除。

---

## 7. 與 Engine 的整合

- Engine `LogicFlowExecutionService` 啟動後每 15 秒 reload 一次 `_scheduleCache`，新增/修改後最多 15 秒生效。
- LogicFlow 流程圖中 A 接點若選「排程模式」並綁定 `scheduleId`，每輪評估時呼叫 `EvalScheduleIsActive(sch)` 取得 ON/OFF 結果。
- Engine 不參與排程 CRUD，純讀取，所以 Web 修改後不需重啟 Engine。

---

## 8. 既有資料相容性

`ExcludeDates`、`IncludeDates` 為新增欄位且 nullable。`DatabaseInitializationService.MigrateColumnsAsync` 啟動時會自動偵測：

```csharp
if (TimeSchedules 表存在 && 沒有 ExcludeDates 欄位)
    ALTER TABLE TimeSchedules ADD ExcludeDates NVARCHAR(MAX) NULL;
if (TimeSchedules 表存在 && 沒有 IncludeDates 欄位)
    ALTER TABLE TimeSchedules ADD IncludeDates NVARCHAR(MAX) NULL;
```

升級後既有資料 NULL，行為與舊版一致。
