# 功能說明書 — 電費報表

> 路由：`/ElectricityCostReport`（EMS 體系，報表查詢選單下）
> 操作流程完全比照[用電報表](功能說明書_能源管理.md)（選迴路 → 粒度時/日/月/年 → 起訖 → 查詢 → 長條圖 + 明細表 + Excel 匯出），查詢結果為**電費（元）**。
> 資料源為電費計算落表的 `ElectricityCostHourly`，見 [功能說明書_電費設定.md](功能說明書_電費設定.md) §電費計算。

---

## 功能總覽

- **查詢條件**：迴路（樹縮排下拉）× 粒度（時/日/月/年）× 起訖 — 與用電報表同一套選擇器與預設值（日粒度預設 = 今天所屬月結期別）
- **月粒度 = 月結期別**：期界由 `BillingPeriodService` 解析，與用電報表 / EMS 電費卡同一語意；月粒度時顯示期別實際區間提示
- **結果**：長條圖（每 bucket 電費，tooltip 附 kWh）+ 明細表（期間 / kWh / 電費雙值欄）+ 合計列
- **無資料引導**：區間內完全無計價資料時顯示「該區間尚無電費資料，請確認已選採用方案並執行重新計算」，不畫全 0 圖
- **Excel 匯出**：格式比照用電報表（查詢條件區 + 表格），欄位 期間 / kWh / 父迴路電費 + 各直接子迴路電費多欄展開；註記列標明「不含基本電費」（+ 估算註記）
- zh-TW / en 雙語；權限管理（ConfigurablePages）可單獨授權（與用電報表分開 — 可授權看度數不看金額）

## 金額計算規則

| 方案型態 | bucket 電費 | 精確性 |
|---|---|---|
| tou / flat | `SUM(ElectricityCostHourly.Cost × EffectiveSign)` 直接加總 | 精確（落表值即真相） |
| progressive | 對每個月結期別取累計 kWh 套級距（夏月 kWh 占比加權）→ 期別總電費按 bucket kWh 占比分攤 | 月粒度 bucket = 期別 → 精確；時/日/年為分攤值（UI/Excel 註記估算） |

- progressive 級距分母 = **根迴路（主要電表，未設定時第一個根節點）**該期別 progressive kWh；子迴路金額 = 根迴路級距金額 × kWh 占比（同 EMS 電費卡語意，`isEstimated` 註記）
- 級距費率取「目前設定中同 PlanId 的方案」（列上有 PlanId 快照；方案已被移除時該部分金額略過）
- 歷史資料若由不同方案計價（未重算），tou/flat 直接加總混方案金額 — 落表即真相；需要一致口徑時至電費設定執行重新計算
- **金額只含流動電費** — 不含基本電費，也不含簡易型 tou 的月總度數超額加價（EMS 電費卡有含超額加價 → 觸發加價時兩處合計會差該金額，屬預期）
- 迴路 kWh/電費依葉子 × EffectiveSign 加總，Sign=-1 迴路為負值（長條圖可為負柱）

## API

| Method | 路徑 | 說明 |
|---|---|---|
| GET | `/ElectricityCostReport/api/circuits` | 迴路樹（flat 清單，同用電報表） |
| POST | `/ElectricityCostReport/api/query` | 查詢 `{ circuitId, granularity, start, end }` → `CostReportResult`（`hasData=false` 時前端顯示引導） |
| POST | `/ElectricityCostReport/api/export` | 匯出 Excel（含直接子迴路多欄；本身是葉子則不展開） |

## 實作備註

- bucket 切界沿用 `EnergyReportService.BuildBoundaries/BuildLabels`（時/日/年）與 `BillingPeriodService.GetPeriodRangesAsync`（月）— 零新切界邏輯
- 查詢粒度為 hour 時按小時列分組，其餘按日彙總列分組（一日必落單一 bucket / 單一期別，全年查詢不會灌爆記憶體）
- 期別歸屬：涵蓋範圍前後各多抓一個月的期別，時刻落入多期（重疊）時取起點最晚者（同 `GetCurrentPeriodAsync` 語意）

## 已知限制

- 需先有計價資料：只涵蓋「選定方案後自動計價 + 重算過」的範圍
- progressive 時/日/年粒度金額為期別級距分攤值（同期別內每度有效單價相同 → 圖形與 kWh 圖等比例）
- 不含基本電費與超額加價（見上）；無用電報表的「明細/合計」堆疊切換與斷線 ⚠ 標記（計價表未保存 staleness 資訊）
