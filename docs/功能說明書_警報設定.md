# 功能說明書：警報設定 (AlarmSetting)

## 1. 功能概述

警報設定頁面提供**警報規則的 CRUD 管理介面**，讓使用者為 Modbus 點位或計算點位設定上限警報、下限警報、DI 狀態警報等三種類型的監控條件。設定完成的規則會由 Engine 端的 `AlarmMonitorService` 即時載入並執行監控判定。

### 核心能力
- **三種警報類型**：上限（AI ≥ 閾值）、下限（AI ≤ 閾值）、DI 狀態（ON/OFF 觸發）
- **四級嚴重度**：緊急 (0)、高 (1)、中 (2)、低 (3)
- **死區 (Deadband)**：防止臨界值附近頻繁觸發
- **UPSERT 語意**：同一 SID 自動覆蓋，無需手動處理重複
- **即列切換啟停**：表格中直接透過 checkbox 快速切換規則啟停
- **跨模組整合**：ScadaPage 與 Realtime 頁面的即時數據會讀取這些規則進行著色

## 4. 資料流架構

```
使用者開啟 /AlarmSetting
    │
    ├─ Controller 查詢 DB ──────────────────────────
    │   ├─ GetAllCoordinatorsAsync()       → 設備清單（Modal 下拉選單）
    │   ├─ GetAllModbusPointsAsync()       → Modbus 點位清單
    │   ├─ GetAllCalculatedPointsAsync()   → 計算點位（合併入點位清單）
    │   └─ AlarmRuleService.GetAllRulesAsync() → 所有規則（JOIN 點位名稱）
    │
    ├─ Razor SSR ──────────────────────────────────
    │   ├─ window._alarmInitData.coordinators  ← JSON 設備陣列
    │   ├─ window._alarmInitData.points        ← JSON 點位陣列
    │   └─ window._alarmInitData.rules         ← JSON 規則陣列
    │
    ├─ JS 初始化 ──────────────────────────────────
    │   ├─ populateCoordinators()  ← 填入設備下拉選單
    │   ├─ renderTable()           ← 渲染規則表格
    │   └─ toggleSection()         ← 初始化各區段顯示/隱藏
    │
    └─ 使用者操作 ─────────────────────────────────
        ├─ 新增規則  → showAddModal() → Modal → save()
        │              → POST /api/alarm-rules → location.reload()
        ├─ 編輯規則  → editRule(id) → Modal → save()
        │              → POST /api/alarm-rules → location.reload()
        ├─ 刪除規則  → deleteRule(id) → confirm()
        │              → DELETE /api/alarm-rules/{id} → location.reload()
        └─ 切換啟停  → toggleEnabled(id, checked)
                       → POST /api/alarm-rules → renderTable()（不重整）

Engine 端（獨立 Worker Service）：
    AlarmMonitorService（每 60 秒重載規則）
        → 讀取 AlarmRules WHERE IsEnabled = 1
        → 比對即時 MQTT 數據
        → 觸發/恢復警報 → 寫入 EventLog
```

## 5. 控制器說明

### AlarmSettingController

#### `Index()` — 頁面入口

查詢所有設備、點位（含啟用的計算點位）、警報規則，透過 `ViewBag` 序列化為 JSON 注入前端。

#### `GetAllRules()` — GET `/api/alarm-rules`

回傳所有規則（不論啟停），供 ScadaPage 等其他頁面即時著色使用。

#### `SaveRule()` — POST `/api/alarm-rules`

接收 `AlarmRuleSaveDto`，委託 `AlarmRuleService.SaveRuleAsync()` 執行 UPSERT。

#### `DeleteRule()` — DELETE `/api/alarm-rules/{id}`

依 ID 刪除規則，不存在回 404。

## 6. 服務層 (AlarmRuleService)

| 方法 | 說明 |
|------|------|
| `GetEnabledRulesAsync()` | 取得啟用規則（Engine AlarmMonitorService 用） |
| `GetAllRulesAsync()` | 取得全部規則，LEFT JOIN ModbusPoints + CalculatedPoints 帶出 `szPointName` |
| `SaveRuleAsync(dto)` | UPSERT：有 ID → UPDATE by ID；無 ID → 檢查 SID 存在則 UPDATE，否則 INSERT |
| `DeleteRuleAsync(id)` | DELETE by ID |

### UPSERT 邏輯

```sql
-- 有 ID（編輯模式）
UPDATE AlarmRules SET ... WHERE Id = @Id

-- 無 ID（新增模式）
IF EXISTS (SELECT 1 FROM AlarmRules WHERE SID = @SID)
    UPDATE AlarmRules SET ... WHERE SID = @SID
ELSE
    INSERT INTO AlarmRules (...) VALUES (...)
```

此設計確保每個 SID 只有一筆規則。

## 8. 前端邏輯 (alarmsetting.js)

以 IIFE 封裝，對外暴露 `window._alarm`。

### 8.1 初始化

1. 從 `window._alarmInitData` 讀取 Razor 注入的設備、點位、規則資料
2. 初始化 Bootstrap Modal 實例
3. 填入設備下拉選單（含「計算點位」選項，`id = -999`）
4. 渲染規則表格
5. 初始化各區段（上限/下限/DI）顯示狀態

### 8.2 設備/點位聯動選取

三層聯動下拉選單：

```
設備 (selCoord)  →  子設備 (selSubDevice, 選填)  →  點位 (selPoint)
```

**流程**：
1. 選擇設備 → `onCoordChange()`
   - 計算點位 (`-999`)：直接顯示 CALC- 點位
   - 多 ModbusID 設備：顯示子設備欄位，填入子設備下拉選單
   - 單 ModbusID 設備：隱藏子設備欄位，直接篩選點位
2. 選擇子設備 → `onSubDeviceChange()`：依 SubModbusID 範圍篩選
3. 選擇點位 → `onPointChange()`：將 SID 寫入隱藏欄位

**編輯模式**：`setupSelectorsForSid(sid)` 從 SID 反推：
- 計算點位 → 選擇「計算點位」設備
- 一般點位 → 依 SID 數值範圍找到 Coordinator，必要時找子設備

### 8.3 規則表格渲染 (`renderTable`)

| 欄位 | 內容 |
|------|------|
| 啟用 | checkbox，直接觸發 `toggleEnabled()` |
| 設備/點位 | `getFullPointLabel()`：設備名稱 / 點位名稱 |
| 上限警報 | `≥ {閾值}` + 嚴重度 Badge，未啟用顯示 `-` |
| 下限警報 | `≤ {閾值}` + 嚴重度 Badge，未啟用顯示 `-` |
| DI 警報 | `{ON/OFF}` + 嚴重度 Badge，未啟用顯示 `-` |
| 備註 | 純文字 |
| 操作 | 編輯按鈕 + 刪除按鈕 |

- 停用的規則整列灰色 (`table-secondary`)

### 8.4 Modal 表單操作

#### 新增 (`showAddModal`)
- 重置所有欄位為預設值
- 上限/下限/DI 三個區段預設關閉

#### 編輯 (`editRule`)
- 從 `_rules` 陣列找到對應規則
- `setupSelectorsForSid()` 自動定位設備/點位
- 填入所有欄位值
- 根據 checkbox 狀態顯示/隱藏對應區段

#### 儲存 (`save`)
1. 驗證 SID 已選取
2. 組裝 `AlarmRuleSaveDto` JSON
3. `POST /api/alarm-rules`
4. 成功 → 關閉 Modal + `location.reload()`
5. 失敗 → `alert()` 顯示錯誤

#### 區段切換 (`toggleSection`)
- 上限：`chkAlarmHigh` → 控制閾值、死區、嚴重度欄位顯示
- 下限：`chkAlarmLow` → 控制閾值、死區、嚴重度欄位顯示
- DI：`chkDiAlarm` → 控制觸發狀態、嚴重度、ON/OFF 標籤欄位顯示

### 8.5 即列啟停切換 (`toggleEnabled`)

- 不開 Modal，直接送 POST 更新整筆規則（isEnabled 取新值，其餘不變）
- 成功後更新 `_rules` 陣列中的值並重新渲染表格（不重整頁面）

## 9. 三種警報類型

### 9.1 上限警報 (High Alarm)

| 設定 | 說明 |
|------|------|
| 閾值 | 當 AI 點位即時值 ≥ 閾值時觸發 |
| 死區 | 觸發後，需降至 (閾值 - 死區) 以下才恢復 |
| 嚴重度 | 0=緊急、1=高、2=中、3=低 |

### 9.2 下限警報 (Low Alarm)

| 設定 | 說明 |
|------|------|
| 閾值 | 當 AI 點位即時值 ≤ 閾值時觸發 |
| 死區 | 觸發後，需升至 (閾值 + 死區) 以上才恢復 |
| 嚴重度 | 同上 |

### 9.3 DI 警報 (Digital Input Alarm)

| 設定 | 說明 |
|------|------|
| 觸發狀態 | `ON` 或 `OFF`，當 DI 點位進入此狀態時觸發 |
| 嚴重度 | 同上 |
| ON/OFF 標籤 | 顯示用標籤文字（目前為唯讀） |

## 13. 跨模組整合

### 與 ScadaPage 的整合
ScadaPage 初始化時呼叫 `GET /api/alarm-rules`，將啟用的規則快取到 `_alarmRuleMap[SID]`。Widget 更新時比對閾值，超限則變色（gauge、rtValue、table）或脈動動畫（diPoint）。

### 與 Engine AlarmMonitorService 的整合
Engine 的 `AlarmMonitorService` 每 60 秒呼叫 `GetEnabledRulesAsync()` 重新載入啟用的規則，比對即時 MQTT 數據，觸發時寫入 `EventLog` 事件記錄。

### 與 Realtime 頁面的整合
Realtime 頁面目前不直接讀取警報規則，但透過 MQTT 品質欄位（STALE、BAD）間接反映異常。
