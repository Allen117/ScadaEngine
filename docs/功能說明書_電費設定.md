# 功能說明書 — 電費設定

> 路由：`/TariffSetting`（EMS 體系，能管設定選單下）
> 完全比照台電「各類電價表」（114 年 10 月 1 日起實施版），涵蓋全部 20 種方案，價目/時段可編輯、可還原台電預設。
> **本版只做設定管理，不做電費計算** — 累計度數/金額卡片顯示 `--`，計算（接主要電表）留待後續版本。

---

## 功能總覽

- **方案選擇**：用電類別（表燈住商 / 低壓電力 / 高壓電力 / 特高壓電力）→ 電價方案 兩層下拉，任一方案可「設為採用方案」
- **三種方案型態，一套統一渲染**：
  - `progressive` 累進級距（表燈非時間電價 3 種）：級距邊界 + 夏月/非夏月每度單價可編輯
  - `flat` 單一費率（低壓非時間電價）：基本電費 + 夏月/非夏月每度單價
  - `tou` 時間電價（其餘 16 種）：基本電費表（按戶/按瓩 × 夏/非夏）+ 流動電費時段表（平日/週六/週日及離峰日 × 尖峰/半尖峰/離峰 × 夏/非夏 → 時間區間組 + 單價）
- **夏月起訖**（月-日）可編輯：表燈/低壓預設 06-01～09-30；高壓/特高壓預設 05-16～10-15；EV 方案 06-01～09-30
- **簡易型加價**：每月總度數超過門檻（預設 2,000 度）每度加價（預設 1.04 元）可編輯
- **區間名稱可編輯**：留空時顯示 i18n 預設名（尖峰/半尖峰/離峰）
- **還原台電預設**：單一方案還原，不影響其他方案
- zh-TW / en 雙語；權限管理（AccountSetting）可單獨授權

## 方案清單（20 種）

| 類別 | 方案 |
|---|---|
| 表燈（住商） | 累進 住宅用 / 非營業用 / 營業用；簡易型 TOU 二段/三段；標準型 TOU 二段/三段 |
| 低壓電力（<100 瓩） | 非時間電價；TOU 二段/三段；電動車充換電設施 |
| 高壓電力（≥100 瓩） | TOU 二段；三段（尖峰固定 / 尖峰可變動）；批次生產；電動車充換電設施（95% 註記） |
| 特高壓電力（≥1,000 瓩） | TOU 二段；三段（尖峰固定 / 尖峰可變動）；批次生產 |

排除「包制電價」（路燈/警報器按戶包月，非電表計度）。

## 資料儲存

| 位置 | 內容 |
|---|---|
| `ScadaEngine.Web/Setting/tariff-taipower-defaults.json` | 台電預設 seed（唯讀，隨程式部署；csproj `CopyToOutputDirectory`）。台電調價時**只改此檔** |
| `SystemSettings`（SettingKey = `electricity_tariff`） | 使用者版整份 JSON（含採用方案 + 全部方案），整份載入整份儲存 |

讀取邏輯（`TariffSettingService.GetConfigAsync`）：

1. DB 無值 → 回 seed（不落 DB，第一次儲存才寫入）
2. DB 有值 → 以 DB 為準；seed 中新增的方案（未來台電新版）以 `szPlanId` 比對自動補齊
3. DB JSON 損毀 → log error 後回 seed（不覆寫 DB）

## 資料模型（JSON 結構）

```jsonc
{
  "szActivePlanId": "hv_tou2",          // 採用方案；空字串 = 尚未選擇
  "plans": [{
    "szPlanId": "hv_tou2",              // 穩定識別碼 = i18n key 後綴（tariff.plan.{id}）
    "szCategory": "lighting|lv|hv|ehv",
    "szType": "progressive|flat|tou",
    "szSummerStart": "05-16",           // 夏月起訖（月-日，含）
    "szSummerEnd": "10-15",
    "szNoteKey": "tariff.note.batch",   // 備註 i18n key，可 null
    "tiers":    [{ "nFrom": 0, "nTo": 120, "dSummer": 1.78, "dNonSummer": 1.78 }],  // progressive
    "flatRate": { "dSummer": 4.08, "dNonSummer": 3.87 },                              // flat
    "baseFees": [{ "szKey": "regular_kw", "szUnit": "kw", "dSummer": 223.6, "dNonSummer": 166.9 }],
    "flowRates": [{ "szDayType": "weekday|sat|sun_offday", "szSeason": "summer|nonsummer",
                    "szPeriod": "peak|semipeak|offpeak", "szName": null,   // null = 用 i18n 預設名
                    "ranges": ["16:00-22:00"], "dPrice": 9.39 }],
    "surcharge": { "nOverKwh": 2000, "dPrice": 1.04 }   // 簡易型限定，可 null
  }]
}
```

- `baseFees.szKey` 固定詞彙（i18n key = `tariff.basefee.{szKey}`）：`per_household` / `per_household_1p` / `per_household_3p` / `device_household` / `device_kw` / `demand_household` / `regular_kw` / `nonsummer_kw` / `semipeak_kw` / `sat_semipeak_kw` / `offpeak_kw`
- `baseFees` 的 `dSummer` / `dNonSummer` 為 `null` 表該季不適用（UI 顯示 —，如非夏月契約無夏月價）
- 時間區間 `"HH:mm-HH:mm"`：訖 `24:00` 合法；**起 > 訖 = 跨午夜**（如 `22:30-07:30`）。UI 上訖時選 `00:00` 代表 `24:00`

## 驗證規則（前後端同規則，後端為準）

- **TOU**：每（日別 × 季節）組的時段聯集必須**覆蓋 00:00–24:00 且互不重疊**（跨午夜拆兩段計）；三日別 × 兩季節共 6 組皆須存在
- **累進**：級距連續（下一級 nFrom = 上一級 nTo + 1，UI 自動接續）、上限 > 下限、僅最後一級可「以上」
- 單價 ≥ 0；加價門檻 > 0；夏月起訖為合法「月-日」
- 儲存單一方案時 `szCategory` / `szType` 以伺服器既有值為準（防前端竄改）

## API

| Method | 路徑 | 說明 |
|---|---|---|
| GET | `/TariffSetting/api/config` | 整份設定（含 seed 補齊） |
| POST | `/TariffSetting/api/plan` | 儲存單一方案（body = TariffPlan） |
| POST | `/TariffSetting/api/active` | 設為採用方案 `{ planId }` |
| POST | `/TariffSetting/api/reset` | 還原單一方案為台電預設 `{ planId }`，回傳還原後方案 |

## i18n 慣例

- 方案/類別/型態/時段別/基本電費項目名稱**不存 JSON**，由識別碼組 i18n key 顯示（`tariff.plan.{szPlanId}` 等）→ 雙語自動切換
- 使用者自訂的區間名稱（`szName`）為 user input，不在 i18n 範圍；留空 = 用 `tariff.period.{szPeriod}` 預設名

## 台電調價 SOP

1. 由台電官網下載新版「各類電價表及計算範例」
2. 更新 `Setting/tariff-taipower-defaults.json` 價目（結構不動）
3. 部署後：未改過的使用者第一次儲存前即看到新價；已自訂的方案不受影響，可按「還原台電預設」取新價

## 後續版本（本版明確不含）

- 累計度數/金額計算：主要電表（`EnergyCircuit.IsMainMeter`）× 月結週期（`BillingPeriodService`）× hourly 預聚合逐時落段計價
- 「尖峰時間可變動」的台電指定 30 日維護介面；離峰日（國定假日）清單維護
- 電價生效日版本化（歷史報表鎖舊價）

## 已知限制

- 訖時 `00:00` 在 UI 上代表 `24:00`（flatpickr 無法輸入 24:00），全日時段顯示為 `00:00-00:00`
- EV 高壓 95% 折算、指定 30 天加價僅以備註呈現，不參與（未來的）計算
- 頁面顯示之台電預設價目以 seed 檔為準（114/10/1 版）；台電若調價需更新 seed
